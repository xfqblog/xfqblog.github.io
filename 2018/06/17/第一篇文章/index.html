小白搭建Hexo+github博客（win 10）

发表于 2018-06-17

—搭建过程中可能会遇到一些问题，可能都不一样，但请坚持下去，使用百度/谷歌寻找答案！！
重要工具：pc，Github，git，node.js
第一步 注册安装，准备环境
①注册Github
登入
Build software better, together
​github.com
图标
，选择右上角的sign up
然后根据注册要求，一步一步填写，用户名也的取好一点，后面你所建立的博客名也与这用户名类似。邮箱最好填写能用的邮箱，因为注册成功需要邮箱激活！
成功的后的样子

如果你对github有兴趣的话，可以read the guide ,如果觉的英文比较累的话，强推使用谷歌翻译使用。

点击 New repository，创建一个存储库

注意Repository name应填写http://username.github.io。比如我的便是xfqblog.github.io,然后点击Create repository。成功后的画面下③④⑤

②安装git
Git for Windows. 国内直接从官网下载比较困难，需要翻墙。这里提供一个国内的下载站，方便网友下载，然后选择安装目录后，一直next，next…………就可以。（只针对小白安装）
https://github.com/waylau/git-for-win
​github.com
③安装Node.js
Node.js
​nodejs.org
图标
然后一直一直next，next………。（只针对小白安装）

第二步 环境配置
①配置SSH
安装完成后，在开始菜单里找到“Git”->“Git Bash”

填写github的用户名，所注册的邮箱

②生成ssh密钥
首先要检查自己电脑上是否现有的 SSH key：

说明这是第一次使用git
接下来生成ssh key

敲入：$ ssh-keygen -t rsa -C &amp;amp;quot;邮件地址@youremail.com&amp;amp;quot;（这里的邮箱地址，输入注册 Github 的邮箱地址，&amp;amp;#39;C&amp;amp;#39;为大写）
输入后，出现….

后回车

在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。

注：id_rsa文件是私钥，要妥善保管，id_rsa.pub是公钥文件。
此时，在用户文件夹（如：C:\Users\阿发）下就会有一个新的文件夹.ssh，里面有刚刚创建的ssh密钥文件id_rsa和id_rsa.pub
最后添加公钥到github

点击用户头像，然后点击显示的Settings(设置)选项；在用户设置栏，点击SSH and GPG keys选项，然后点击New SSH key(新建SSH)按钮；将id_rsa.pub中的内容复制到Key文本框中，然后点击Add SSH key(添加SSH)按钮；

测试git是否与github连接

敲入：$ ssh -T git@github.com
后敲入:yes.

出现警告，可以忽视
③安装hexo
在自己电脑I盘（每个人的各不相同）目录下新建文件夹 blog，按住点击鼠标右键git bash here（或者前敲入：$ mkdir blog ，后$ cd blog）

然后敲入：$ npm install -g hexo-cli（win10会出现问题）

确认一下 Node是否 安装正确
解决方式：
$ npm install -g cnpm –registry=https://registry.npm.taobao.org
$ cnpm install hexo-cli -g
注：目前国内npm源有问题；所以键入如下代码即可安装成功。
安装成功后：敲入：$ hexo -v

表示hexo安装成功
初始化hexo
$ hexo init

初始化成功
安装依赖包
$ npm install

安装成功
启动hexo
$ hexo server($hexo s)
